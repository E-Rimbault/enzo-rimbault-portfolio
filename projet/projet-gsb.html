Projet GSB - Application Web avec Laravel

Description Générale

Ce projet a été réalisé dans le cadre du BTS SIO, pour répondre aux besoins du laboratoire Galaxy Swiss Bourdin (GSB), issu de la fusion de plusieurs laboratoires pharmaceutiques. L’objectif était de concevoir une application web permettant aux visiteurs médicaux de centraliser et consulter les rapports de leurs visites, ainsi que les informations sur les praticiens et les médicaments.

Utilité de l'application

L’application GSB a pour but de faciliter le travail des visiteurs médicaux. Elle permet de :

  • 🧾 Ajouter et consulter les rapports de visite rédigés après chaque rencontre avec un praticien.
  • 💊 Accéder à une base de données des médicaments produits par le laboratoire, avec descriptions détaillées.
  • 🩺 Rechercher et consulter les informations sur les praticiens selon leur nom ou spécialité.
  • 👤 Gérer son profil personnel pour maintenir ses coordonnées à jour.

Grâce à cette centralisation, le suivi des visites est plus clair, plus structuré, et permet au laboratoire d’avoir un retour sur les échanges entre les visiteurs et les professionnels de santé.

Technologies Utilisées

  • Laravel : framework PHP pour la structure MVC (modèle-vue-contrôleur).
  • Blade : moteur de templates Laravel pour la création des vues.
  • MySQL : base de données relationnelle pour stocker les utilisateurs, rapports, médicaments, praticiens.
  • Bootstrap : stylisation de l’interface pour un rendu responsive et moderne.
  • PHP Artisan : génération des modèles, contrôleurs, et migrations.

Structure et Codage

L'application repose sur une architecture MVC. Voici comment elle a été structurée :

  • 📂 Models : Chaque entité (Visiteur, Rapport, Praticien, Médicament) possède un modèle Eloquent relié à la base de données.
  • 📂 Controllers : Ils gèrent les actions métiers : affichage, ajout, modification, suppression des données.
  • 📂 Views : Les vues Blade affichent dynamiquement les données récupérées.
  • 🔐 Authentification : système Laravel intégré avec middleware pour sécuriser les routes.
  • 🛠️ php artisan migrate et php artisan make: pour la création de migrations et composants.

Ressources numériques mises à disposition

Dans le cadre de ce projet, plusieurs ressources numériques ont été développées ou mises en œuvre pour répondre aux besoins des utilisateurs finaux :

  • 🔐 Authentification sécurisée : accès restreint par identifiant et mot de passe crypté.
  • 🧾 Formulaire de saisie de compte-rendu : interface dynamique pour enregistrer les visites médicales.
  • 📁 Archivage et historique : chaque compte-rendu est numéroté et consultable à tout moment.
  • 🩺 Recherche de praticiens : moteur de recherche avec affichage des détails.
  • 💊 Base de données des médicaments : accès aux informations thérapeutiques et posologies.
  • 📋 Consultation et édition de profil : chaque utilisateur peut modifier ses informations personnelles.
  • 📊 Visualisation et suivi des rapports : interface claire pour consulter l’historique des visites.

Ces ressources numériques contribuent à la digitalisation des processus métier et améliorent considérablement la gestion des informations au sein du laboratoire GSB.
