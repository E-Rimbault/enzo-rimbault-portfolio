📌 Compétence : Traiter des demandes concernant les applications

Dans le cadre d’un projet encadré par Mr CHOIRAL, plusieurs étapes techniques ont été réalisées en réponse à des demandes précises liées à la sécurisation d’une application web. Ce projet a été conçu pour répondre à des besoins progressifs autour de la gestion des utilisateurs et de l’authentification sécurisée.

✅ Étape 1 : Reproduction de l’authentification de base

  • 🔧 Objectif : Reproduire une page de connexion fonctionnelle avec base de données.
  • 🧩 Utilisation de PHP & MySQL pour gérer l’inscription et la connexion des utilisateurs.
  • 🔐 Hashage sécurisé des mots de passe avec password_hash().
  • 🧪 Vérification à la connexion avec password_verify().

✅ Étape 2 : Journal d'événements - Page log.php

  • 🗂️ Création d'une table logs contenant : identifiant, résultat, IP, date et heure.
  • 🔒 L'accès à cette page est restreint aux utilisateurs authentifiés (via session).
  • 📊 Visualisation sous forme de tableau des connexions réussies ou échouées.

✅ Étape 3 : Double authentification par e-mail (2FA Email)

  • 📧 Envoi d’un code à usage unique par mail après la saisie correcte du mot de passe.
  • ⏳ Code valide uniquement pendant 5 minutes (géré via sessions PHP).
  • 🔁 Redirection vers la page d’accueil si le code est correct, sinon retour à l’écran de connexion.

✅ Étape 4 : Double authentification avec Google Authenticator (TOTP)

  • 📱 QR code généré à l’inscription avec une clé secrète personnelle (TOTP).
  • 🔐 Vérification du code temporaire via l’algorithme TOTP côté serveur.
  • 📲 Application compatible : Google Authenticator, Microsoft Authenticator, etc.
